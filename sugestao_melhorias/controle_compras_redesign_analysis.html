Análise de UI/UX para redesign do ControleCompras

Problemas críticos identificados
Tipografia Alta prioridade
Atual
Font padrão do sistema, sem hierarquia visual clara, MAIÚSCULAS em labels como "E-MAIL", "SENHA" — cria ruído e parece formulário anos 2000
Sugestão
Inter (Google Fonts) — padrão de apps fintech modernos. Labels em sentence case, peso 400/500/600. Títulos de seção em 11px uppercase com letter-spacing, não os campos.
Layout responsivo Alta prioridade
O dashboard usa dois painéis side-by-side fixos sem breakpoints claros. Em tablets/mobile o formulário "Registrar Compra" e "Filtros" ficam espremidos. A tabela de registros não tem scroll horizontal — quebrará em telas menores.
Atual
Grid fixo 2 colunas, sem media queries efetivas, tabela sem overflow-x: auto
Sugestão
CSS Grid com grid-template-columns: repeat(auto-fit, minmax(320px, 1fr)). Mobile-first: formulário empilha acima dos filtros. Tabela em wrapper com scroll.
Paleta de cores Média prioridade
Roxo vibrante (#7c3aed aprox.) como CTA principal funciona, mas os cards de stats usam fundos muito escuros e baixo contraste entre texto secundário e fundo. O fundo do app é quase preto puro — cansativo para uso prolongado.

Formulário de entrada Média prioridade
Campos sem ícones de apoio visual, sem estados de erro visíveis, sem feedback de validação inline. O botão "Salvar Compra" não tem estado de loading após submit.
Atual
Input simples, sem prefix icon, sem hint text, sem validação em tempo real
Sugestão
Prefix icon nos inputs (R$ no valor, calendário na data), hint text abaixo do campo, border-color muda para verde/vermelho on validate, botão vira spinner no submit
Melhorias de UX / interação
Gráfico de distribuição Melhoria
O donut chart atual é funcional mas genérico. Adicionar: hover com tooltip detalhado, animação de entrada (stroke-dasharray de 0 ao valor), legenda com porcentagem calculada dinamicamente ao mudar filtro. Considerar um chart de barras horizontais como alternativa — mais legível em mobile.
Tabela de registros Melhoria
Sem paginação, sem ordenação por coluna, sem busca inline. Em mobile, substituir tabela por cards empilhados (cada compra vira um card com ícone da categoria, valor em destaque e ações inline).
Feedback de ações Melhoria
Adicionar toast notifications (bottom-right, auto-dismiss em 3s) para confirmar salvar/deletar compras. Usar biblioteca como vue-toastification ou implementar com Teleport nativo do Vue 3.
Roadmap de implementação sugerido
1
Tipografia — importar Inter via @fontsource/inter no Vite, aplicar no CSS global. Normalizar labels para sentence case. 30min de trabalho.
2
Responsividade — refatorar o grid do dashboard com CSS Grid auto-fit. Adicionar breakpoint mobile-first em 640px. Encapsular tabela em <div style="overflow-x:auto">.
3
Paleta refinada — ajustar CSS variables no :root. Usar --bg-primary: #1a1b2e, --bg-surface: #1e2035, --bg-card: #252641 para criar profundidade com múltiplos níveis de superfície.
4
Micro-interações — hover states nos botões de ação da tabela (escala 1.1 no ícone), transição smooth no dropdown de categoria, animação no donut chart ao carregar dados.
5
Mobile table → cards — usar v-if="isMobile" computed com window.innerWidth < 640 para renderizar cards no lugar da tabela em telas pequenas.
